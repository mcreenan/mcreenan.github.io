Create `src/components/ThemeDialog.astro` as a native `<dialog>` element with radio inputs for theme selection.

**Component structure (per UI-SPEC.md):**

The Astro component has an empty frontmatter block (no props needed) followed by the template. The template contains:

1. A `<dialog id="theme-dialog" class="theme-dialog">` element
2. Inside it, a `<div class="theme-dialog-content">` wrapper
3. A `<div class="theme-dialog-title">Select Theme</div>` heading
4. A `<div class="theme-dialog-options">` containing 3 labels, each with:
   - `<label class="theme-option">` wrapping an `<input type="radio" name="theme" value="retro|win31|win95" />` and a `<span>` with the label text ("Retro", "Win 3.1", "Win 95")
5. A `<button class="button theme-dialog-close" type="button">Apply Theme</button>`

Then a `<script>` tag (NOT is:inline, this is a bundled script) containing:

- `initThemeDialog()` function that:
  - Gets the dialog element by id `theme-dialog`
  - Queries all `input[name="theme"]` radio inputs
  - Pre-selects the current theme from `localStorage.getItem('theme') || 'retro'` (SWITCH-02)
  - Adds `change` event listener to each radio that calls `localStorage.setItem('theme', r.value)` AND `document.documentElement.setAttribute('data-theme', r.value)` for instant apply (D-06)
  - Adds click listener to `.theme-dialog-close` button that calls `dialog.close()`
  - Adds click listener to the dialog itself for click-outside-to-close: `if (e.target === dialog) dialog.close()`

- Event listener: `document.addEventListener('astro:after-swap', initThemeDialog)` for View Transitions reinit
- Initial call: `initThemeDialog()`

- Global function exposure: `(window as any).openThemeDialog = function()` that:
  - Gets dialog by id
  - Pre-selects current theme radio from localStorage
  - Calls `dialog.showModal()` (NOT `dialog.show()` — modal is required for focus trap)

**Keyboard behavior (D-07, SWITCH-03):** No custom keyboard handlers needed. Native `<dialog>` with `showModal()` provides focus trap, Escape-to-close, and Tab navigation. Native radio inputs handle arrow-key cycling.
