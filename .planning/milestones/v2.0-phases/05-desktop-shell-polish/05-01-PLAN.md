  <action>
Add a `<script>` tag (Astro bundled, NOT is:inline) at the bottom of index.astro, inside the `<body>` tag, before the closing `</body>`. This script implements the full overlay controller.

The script must implement these functions:

**1. initDesktopOverlay()** — Gate on theme, attach click handlers:
- Read theme from `document.documentElement.getAttribute('data-theme')`
- If theme is NOT 'win31' and NOT 'win95', return immediately (per D-02, SHELL-03 — retro uses page nav)
- Query all `.pm-icon` and `.win95-icon` anchor elements
- For each, add click listener that calls `event.preventDefault()` AND `event.stopPropagation()` (stopPropagation prevents Astro View Transitions from intercepting — per RESEARCH Pitfall 1), then calls `openOverlay(anchor.getAttribute('href'), anchor)`

**2. Content cache** — A `Map` object keyed by URL path:
- `const contentCache = new Map()` at module scope
- Stores `{ title: string, html: string }` after first fetch
- Avoids re-fetching static content (per RESEARCH anti-patterns)

**3. openOverlay(url, triggerEl)** — Fetch, extract, display, pushState:
- Store triggerEl in module-scoped variable `activeTrigger`
- Set module-scoped `overlayIsOpen = true`
- If url NOT in contentCache: fetch(url), get response text, parse with `new DOMParser().parseFromString(html, 'text/html')`, extract content:
  - Title: Use a static lookup map `{ '/about': 'About Me', '/work': 'Work', '/contact': 'Contact' }` (per RESEARCH Open Question 3 recommendation — avoids selector fragility)
  - Body HTML: Extract from `.window-content` for the fetched doc (win31 content area) OR `.window-body` (win95 content area). Use `.window-content` as primary — it exists in both theme variants on subpages. If not found, try `.window-body`. Store the `.innerHTML` string.
  - Cache result in contentCache
- Get the overlay container: `document.getElementById('overlay-window')`
- Set both `.overlay-title` elements' textContent to the cached title
- Set both `.overlay-body` elements' innerHTML to the cached body HTML
- Remove the `hidden` attribute from the overlay container
- Call `history.pushState({ overlayUrl: url }, '', url)` (per D-03)
- Focus the first focusable element inside the visible overlay chrome (the close button)

**4. closeOverlay(fromPopstate = false)** — Hide overlay, restore focus, manage history:
- If NOT overlayIsOpen, return
- Set overlayIsOpen = false
- Add `hidden` attribute back to overlay container
- Clear overlay-body innerHTML (free memory)
- If `fromPopstate` is false: call `history.back()` (user initiated close — Escape/click/button). If fromPopstate is true: do NOT call history.back() (per RESEARCH Pattern 5 critical nuance — popstate already changed state)
- Call `activeTrigger?.focus()` to return focus to the icon that opened the overlay (per D-07, RESEARCH Pitfall 3)
- Set activeTrigger = null

**5. Focus trap** — Keyboard handler on overlay container:
- Add keydown listener to the overlay container
- On Escape key: call `closeOverlay()` (per D-04)
- On Tab key: implement Tab/Shift+Tab cycling within focusable elements inside overlay (per D-07). Use the FOCUSABLE selector: `'a[href], button:not([disabled]), [tabindex]:not([tabindex="-1"])'`. Get all matching elements in the overlay. If Shift+Tab on first element, wrap to last. If Tab on last element, wrap to first. Call `event.preventDefault()` in both wrap cases.

**6. Backdrop click handler** — Click outside closes overlay:
- Add click listener on `.overlay-backdrop` that calls `closeOverlay()` (per D-04)

**7. Close button handlers** — Wire both close buttons:
- Add click listeners to all `.overlay-close` buttons that call `closeOverlay()` (per D-04)

**8. popstate handler** — Back button closes overlay:
- Add `window.addEventListener('popstate', ...)` handler
- Guard: `if (!overlayIsOpen) return` (per RESEARCH Pitfall 4)
- Call `closeOverlay(true)` (fromPopstate = true)

**9. astro:after-swap re-initialization** — Required for View Transitions:
- `document.addEventListener('astro:after-swap', () => { overlayIsOpen = false; contentCache.clear(); initDesktopOverlay(); })`
- Call `initDesktopOverlay()` immediately for first load

**Important implementation notes:**
- The script tag should be `<script>` (Astro bundled/deduped), NOT `<script is:inline>`. Bundled scripts are deduped across navigations and support module scope.
- All module-scoped variables (contentCache, overlayIsOpen, activeTrigger) persist within the script module scope.
- The `initDesktopOverlay` function must re-query DOM elements each time it runs (after View Transition, DOM is replaced).
- Event listeners on `#overlay-window` and `window` (popstate) should be attached once at module scope, NOT inside initDesktopOverlay (they survive DOM replacement since #overlay-window is on index.astro only).
