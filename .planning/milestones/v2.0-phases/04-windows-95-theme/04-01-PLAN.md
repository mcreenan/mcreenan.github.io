2. The current Window.astro has a single markup block with win31 class names. Restructure so the component renders DIFFERENT markup based on variant. 3. Add a CONDITIONAL win95 branch that renders ONLY when `variant === "win95"`. This branch uses 98.css class names (`.title-bar`, `.title-bar-text`, `.title-bar-controls`, `.window-body`) which are different from the retro/win31 names:

```astro
{variant === "win95" ? (
  <div
    class={`window ${contentWindow ? "content-window" : "splash-screen"}`}
    style={`height: ${height};`}
  >
    {title && (
      <div class="title-bar">
        <div class="title-bar-text">{title}</div>
        <div class="title-bar-controls">
          <button aria-label="Minimize" type="button"></button>
          <button aria-label="Maximize" type="button"></button>
          {contentWindow ? (
            <button aria-label="Close" type="button" onclick="window.location='/'"></button>
          ) : (
            <button aria-label="Close" type="button"></button>
          )}
        </div>
      </div>
    )}
    <div class="window-body">
      <slot />
    </div>
  </div>
) : (
  /* Existing retro/win31 markup — keep exactly as-is */
  <div
    class={`window fixed splash-screen ${contentWindow ? "content-window" : ""}`}
    style={`height: ${height};`}
  >
    {title && (
      <div class="window-titlebar">
        <div class="win31-titlebar-left">
          {contentWindow ? (
            <a href="/" class="win31-chrome-btn" aria-label="Close - return to home">-</a>
          ) : (
            <span class="win31-chrome-btn" aria-hidden="true">-</span>
          )}
        </div>
        <span class="window-title">{title}</span>
        <div class="win31-titlebar-right">
          <span class="win31-chrome-btn win31-btn-raised" aria-hidden="true">&#9650;</span>
          <span class="win31-chrome-btn win31-btn-raised" aria-hidden="true">&#9660;</span>
        </div>
      </div>
    )}
    <div class="window-inner">
      <div class="window-content">
        <slot />
      </div>
    </div>
  </div>
)}
```

Key points per D-11:
- Close button uses `onclick="window.location='/'"` (not `<a>`) to keep 98.css `button[aria-label="Close"]` selector matching for the X icon (per RESEARCH.md Pitfall 3).
- Minimize and Maximize are decorative — `type="button"` prevents form submission, no onclick.
- The `window-body` class (98.css) replaces `window-inner` + `window-content` for win95 only.
- contentWindow pages get the close-as-home-link; non-contentWindow (index splash) gets a plain close button.
