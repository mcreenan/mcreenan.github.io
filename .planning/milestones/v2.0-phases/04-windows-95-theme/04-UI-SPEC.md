## Scope Constraint

This phase adds `[data-theme="win95"]` CSS overrides only. All existing `[data-theme="retro"]` and `[data-theme="win31"]` rules remain untouched. The `.window` class collision is resolved by the scoping pattern already established in Phase 2 (global.css line 457: `[data-theme="retro"] .window`).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon label gaps, title bar padding |
| sm | 8px | Button padding, menu item padding, icon grid gap |
| md | 16px | Icon grid padding, window content padding |
| lg | 24px | Desktop icon grid spacing |
| xl | 32px | Layout gaps |
| 2xl | 48px | Desktop icon size (width and height) |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Taskbar height: 28px (Win95 reference — not a multiple of 4 but period-accurate). Fixed at bottom with `position: fixed; bottom: 0`.
- Win95 title bar height: 18px (98.css default, period-accurate).
- Window three-button cluster: each button 16x14px (Win95 chrome proportions per 98.css spec).
- Touch target minimum: 44px height for taskbar Start button area (accessibility floor).

**Source:** CONTEXT.md D-07 (48x48 icons), D-05 (taskbar); Win95 reference proportions.

## 98.css Integration Contract

**Scope:** All 98.css rules imported and wrapped under `[data-theme="win95"]` selector. Method is CSS `@layer` or manual prefix — executor decision per CONTEXT.md discretion list. No 98.css rules may leak to global scope.

**Classes activated by 98.css (under win95 scope):**
- `.window` — beveled 3D window chrome
- `.title-bar` — gradient title bar with text
- `.title-bar-controls button` — min/max/close button cluster
- `button` — beveled button style
- `.sunken-panel` — inset content area (scrollbar region)

**Collision safety:** Phase 2 already scoped `[data-theme="retro"] .window`. Phase 3 scoped `[data-theme="win31"] .window`. Win95 follows the same pattern — existing theme selectors are unaffected.

**Source:** CONTEXT.md D-01, D-02, D-03, STATE.md research flag for .window collision.

## Component Contracts

### Window.astro — win95 variant

Add `variant="win95"` to the existing `"retro" | "win31"` union.

Win95 markup uses 98.css class names inside the variant branch:

```
<div class="window">               ← 98.css beveled chrome
  <div class="title-bar">          ← gradient bar
    <div class="title-bar-text">{title}</div>
    <div class="title-bar-controls">
      <button aria-label="Minimize"></button>   ← decorative (no action)
      <button aria-label="Maximize"></button>   ← decorative (no action)
      <button aria-label="Close" onclick="...or href='/'"></button>  ← links home on subpages
    </div>
  </div>
  <div class="window-body">        ← 98.css content area
    <slot />
  </div>
</div>
```

Close button behavior: links home (mirrors Win31 `[-]` home-link pattern from Phase 3, CONTEXT.md D-11).

Minimize and Maximize: decorative only, no JS action. No `aria-label` fabrication — use 98.css standard labels.

### Taskbar Component (new)

Structure:
```
<div class="win95-taskbar">
  <button class="win95-start-btn">  ← Start button (decorative, no menu)
    [Win flag SVG or inline icon]
    Start
  </button>
  <div class="win95-tray">
    <time class="win95-clock" id="win95-clock">12:34 PM</time>
  </div>
</div>
```

Positioning: `position: fixed; bottom: 0; left: 0; right: 0; height: 28px; z-index: 9999`.

Clock JS: `setInterval` updating every 60 seconds, formatted as `hh:mm AM/PM` (12-hour, no seconds). Inline `<script>` in Taskbar component.

Visibility: Hidden by default, shown only under `[data-theme="win95"]` via CSS display toggle (matches the `.program-manager` pattern from Phase 3).

**Source:** CONTEXT.md D-05, D-06. Taskbar height 28px (Win95 reference).

### Desktop Icons — win95 layout

Desktop icon grid for Win95 home page (alongside `.retro-layout` and `.program-manager` toggles):

```
.win95-desktop: display: none (default)
[data-theme="win95"] .win95-desktop: display: flex / grid
[data-theme="win95"] .retro-layout: display: none
[data-theme="win95"] .program-manager: display: none
```

Icon specs:
- Size: 48x48px (CONTEXT.md D-07)
- Placement: centered grid on desktop area above taskbar
- Icon pixel art: 16-color Win95 palette, `image-rendering: pixelated`
- Label: 12px W95FA, white text, black stroke, centered below icon
- Hover/focus: `background: rgba(0, 0, 128, 0.5)` selection highlight with dotted outline

Three icons required: About (person), Work (briefcase), Contact (envelope). Scale up the Phase 3 32x32 SVGs to 48x48 viewBox, or redraw at native 48px grid.

**Source:** CONTEXT.md D-07, D-08. Phase 3 ProgramManager.astro established SVG pixel art pattern with 16-color palette.

### ThemesIcon / Theme Access

Win95 theme access: right-click context menu on desktop OR Start button area (CONTEXT.md D-10). Implementation detail is executor discretion. The existing `window.openThemeDialog()` global call is the integration point — same as Phase 3's `menubar-dropdown button onclick`.

## Interaction Contract

### Clock
- Initializes on DOMContentLoaded, updates every 60 seconds via `setInterval`
- Formats using `Date` object: 12-hour clock, no seconds, space-separated AM/PM
- Graceful static fallback if JS fails (pre-rendered server-side time string in the `<time>` element)

### Desktop Icons
- Click: navigate to `/about`, `/work`, `/contact` (anchor `<a>` elements, not buttons)
- Hover: `background: rgba(0, 0, 128, 0.5)` highlight
- Focus: same highlight + `outline: 1px dotted #FFFFFF; outline-offset: -1px` (Win95 pattern)

### Start Button
- Decorative in this phase — no Start menu opens (CONTEXT.md deferred)
- Still a `<button>` element (not `<div>`) for keyboard tab order
- No `onclick` action — cursor default

### Window Close Button (subpages)
- `<a href="/">` wrapped around the 98.css close button, or `onclick="window.location='/'"`
- Returns user to home page, matching Win31 `[-]` behavior

### Theme Dialog Trigger
- Win95 equivalent: right-click on desktop surface or "Change Theme..." in Start menu area
- Calls `window.openThemeDialog()` — same global already wired in Phase 2

### No Drag/Resize
- Windows are static positioned — no draggable chrome (explicitly Out of Scope in REQUIREMENTS.md)

## Animation Contract

- No CSS transitions on theme switch (consistent with existing themes — Phase 5 may add THEME-01)
- No hover animations on Win95 buttons (98.css handles bevel state via box-shadow swap, not transforms)
- Win95 window appears at fixed position — no open/close animation
- `prefers-reduced-motion`: no animations to reduce in this phase

## Accessibility Contract

- Taskbar clock: wrapped in `<time>` element with `datetime` attribute (ISO format)
- Desktop icons: `<a>` elements with visible text labels (no aria-label needed — label IS the text)
- Start button: `<button>` with `type="button"`, visible "Start" text label
- Win95 window buttons: 98.css provides `aria-label` attributes via standard HTML — do not remove them
- Focus ring: Win95 desktop icon focus state uses dotted outline matching Win95 selection visual
- Keyboard tab order: taskbar Start button is focusable; desktop icons are in document flow
