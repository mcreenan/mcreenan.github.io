  <action>
Add a `<script is:inline>` block to BaseHead.astro that runs BEFORE any stylesheet is applied. This script must appear in the template output BEFORE the `<ViewTransitions />` and any stylesheet `<link>` tags.

Place this as the FIRST element in the template output of BaseHead.astro (before the `<!-- Global Metadata -->` comment):

```astro
<script is:inline>
  (function() {
    var saved = localStorage.getItem('theme');
    document.documentElement.setAttribute('data-theme', saved || 'retro');
  })();
</script>
```

**Critical constraints:**
- MUST use `is:inline` directive — without it, Astro bundles the script as a module (deferred), which defeats the purpose (per Pitfall 1 in RESEARCH.md)
- MUST use `var` not `const`/`let` — inline scripts run in global scope; var is safer for IIFE cleanup
- MUST use IIFE to avoid polluting global scope
- MUST default to `'retro'` when no saved theme exists (per D-12)
- MUST set on `document.documentElement` (not `document.body`) because View Transitions replaces body (per Pitfall 2)

The global.css import in the frontmatter (`import "../styles/global.css"`) generates a `<link>` tag. The `is:inline` script in the template will appear before it in the HTML output because Astro emits frontmatter-generated stylesheets after template content in `<head>`.

  <action>
Add a SEPARATE regular (bundled) `<script>` tag to BaseHead.astro for View Transitions theme reapplication. This goes AFTER the is:inline script, somewhere in the template body (position relative to other elements does not matter for bundled scripts — they are deferred):

```astro
<script>
  function applyTheme() {
    const theme = localStorage.getItem('theme') || 'retro';
    document.documentElement.setAttribute('data-theme', theme);
  }

  document.addEventListener('astro:after-swap', applyTheme);
  applyTheme();
</script>
```

**Why a separate script:** The is:inline script handles first paint. This bundled script handles View Transitions navigation. They serve different purposes:
- `is:inline`: Runs synchronously before paint, prevents flash on initial load
- Bundled script: Registers the `astro:after-swap` event listener for navigation, and calls `applyTheme()` as a fallback on initial load

**Do NOT add is:inline to this script** — it does not need to be render-blocking, and making it inline would prevent Astro from deduplicating it across pages.

This script also serves as the foundation that Plan 03's ThemeDialog will use to update the theme — when the dialog sets localStorage and calls setAttribute, the next navigation will pick it up via this listener.
