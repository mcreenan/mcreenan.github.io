**Part 2 — Replace hardcoded values with var() references in ALL rules:**

Scope theme-dependent rules under `[data-theme="retro"]` to prevent future collision with 98.css (per Pitfall 5 in RESEARCH.md and CSS Architecture Contract in UI-SPEC.md). Rules that are purely structural (dimensions, display, flex, position) stay unscoped.

Key replacements:
- `body`: `background-color: var(--color-bg)`, `font-family: var(--font-body)`, background-image rgba values replaced with `var(--bg-gradient-fade)`, `var(--bg-pixel-light)`, `var(--bg-pixel-dark)`
- `.window`: scope under `[data-theme="retro"]`, use `background-color: var(--color-window)`
- `.button`: `background-color: var(--color-button-bg)`, `border: var(--border-button)`, `box-shadow: var(--shadow-button)`, `color: var(--color-button-text)`
- `.button:hover`: `box-shadow: var(--shadow-button-hover)`
- `.os-title`: `font-family: var(--font-title)`, `color: var(--color-title)`
- `.splash-screen`: `background: var(--bg-splash)`, `box-shadow: var(--shadow-window)` — keep `@keyframes borderGradient` and `animation`/`border-image-source` hardcoded (D-09, retro-only)
- `.headshot-image`: `border: var(--border-headshot)`, `box-shadow: var(--shadow-headshot)`
- `.content-window`: `background: var(--bg-content-window)`
- `.window-content::-webkit-scrollbar-track`: `background: var(--color-scrollbar-track)`, `border: var(--border-scrollbar)`
- `.window-content::-webkit-scrollbar-thumb`: `background: var(--color-scrollbar-thumb)`
- `.window-content::-webkit-scrollbar-thumb:hover`: `background: var(--color-scrollbar-thumb-hover)`
- `.window-title`: `background: linear-gradient(90deg, var(--color-title-gradient-start), var(--color-title-gradient-end))`, `font-family: var(--font-title)`, text-shadow values use `var(--color-title-shadow)`, `var(--color-title-glow-1)`, `var(--color-title-glow-2)`
- `.desktop-icon`: `color: var(--color-desktop-icon)`, `text-shadow: 1px 1px 0 var(--color-desktop-icon-shadow)`
- `.desktop-icon:hover`: `background: var(--color-desktop-icon-highlight)`
- `.desktop-icon:focus`: `background: var(--color-desktop-icon-highlight)`, `outline: var(--outline-desktop-icon)`
- `.desktop-icon span`: text-shadow uses `var(--color-desktop-label-stroke)` in all 4 directions
- `.desktop-icon i`: `filter: var(--filter-desktop-icon)`
- `.window-content a:not(.button):not(.desktop-icon)`: `color: var(--color-link)`
- `.window-content a:not(.button):not(.desktop-icon):hover`: `color: var(--color-link-hover)`

**IMPORTANT:** The `body` rule's `background-image` uses the teal color `rgba(0, 128, 128, 0)` in the linear-gradient. This is the transparent version of the bg color for the gradient fade effect. Keep the gradient structure but replace the colored rgba values with token references where they map to tokens. The base teal transparent references (`rgba(0, 128, 128, 0)`) are structural to the gradient and stay as-is since they are just "transparent" anchors.

**NOT to tokenize (per D-09):** `@keyframes borderGradient` colors and `.splash-screen` `border-image-source` / `animation` properties remain hardcoded. Scope them under `[data-theme="retro"]` since they are retro-specific.

**NOT to tokenize (structural):** All width, height, padding, margin, display, flex, position, overflow, z-index, transform, cursor, transition values. These are layout, not theme.

The `.os-title` text-shadow value `2px 2px 0 rgba(0, 0, 0, 0.2)` is a structural shadow (black alpha), not the themed title shadow — keep it or tokenize as `--shadow-os-title` at your discretion.

Media query rules at 640px and 380px: only re-scope theme-colored properties under `[data-theme="retro"]`. Dimensional overrides (width, height, padding, font-size) for responsive layout stay unscoped.
