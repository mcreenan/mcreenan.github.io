## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Title bar internal padding: `padding: 2px 4px` on `.window-titlebar` |
| sm | 8px | Button internal padding, menu bar item padding: `padding: 2px 8px` on `.menubar-item` |
| md | 16px | Window content padding: `padding: 1rem` on `.window-content` |
| lg | 24px | Program Manager icon group gaps |
| xl | 32px | Program Manager window outer margin |
| 2xl | 48px | Desktop vertical centering offset |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Title bar height: 18px (Win 3.1 system metric — not on scale; period accuracy takes priority)
- Icon size: 32x32px (Win 3.1 standard icon size — fixed metric per D-02)
- Menu bar item padding: `2px 6px` (period-accurate tightness, not on 8pt scale)
- Button touch target: minimum 44px height (accessibility floor — overrides visual period accuracy on mobile)

Source: CONTEXT.md D-09, D-02; RESEARCH.md Pattern 2

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Chrome (title bars, buttons, menu bar) | 11px | 400 (normal) | 1.0 | W95FA, Arial, sans-serif |
| Body / window content | 11px | 400 (normal) | 1.5 | W95FA, Arial, sans-serif |
| Desktop icon label | 11px | 400 (normal) | 1.2 | W95FA, Arial, sans-serif |
| Program Manager title ("Matt Creenan") | 11px | 400 (normal) | 1.0 | W95FA, Arial, sans-serif |

Notes:
- W95FA is a bitmap font designed for 8px, 10px, and 11px. Use 11px for chrome elements, 11px for body. Do NOT use rem/em for chrome — bitmap fonts require integer pixel sizes.
- Font rendering: set `-webkit-font-smoothing: none` on `[data-theme="win31"]` chrome elements. Accept that Windows/Chrome ClearType may still apply smoothing — this is a known limitation per RESEARCH.md.
- font-display: swap — graceful fallback to Arial, sans-serif per D-17.
- Two weights maximum: 400 (normal) for all chrome; no bold used in Win 3.1 UI chrome. Body content may use bold inline for emphasis.

Source: CONTEXT.md D-15, D-16, D-17, D-18; RESEARCH.md Pitfall 2

## Win 3.1 Component Contracts

### Window Chrome (WIN31-01)

- Outer border: `2px solid #000000`
- Window background: `#C0C0C0`
- Box shadow: `none` — strictly flat, no drop shadows (D-11)
- No gradients anywhere — override all retro gradient tokens with solid colors
- Title bar: solid navy `#000080`, height 18px, white `#FFFFFF` 11px W95FA text
- Title bar layout: flex row — `[-]` control box (left) + title text (center) + `[^][v]` (right)
- All title bar chrome buttons are decorative `<span>` elements — not interactive, no `role="button"`, `aria-hidden="true"`
- Retro margin hack override required: `.splash-screen` must have `border-top-width: 2px` (not 21px) and `.window-titlebar` must have `margin: 0`
- Retro title transform override: `[data-theme="win31"] .window-title { transform: none; background: none; -webkit-background-clip: unset; color: #FFFFFF; font-size: 11px; text-shadow: none; }`

Source: CONTEXT.md D-09, D-10, D-11; RESEARCH.md Pattern 2, Pitfall 1

### Button Style (WIN31-02)

- Background: `#C0C0C0`
- Border: `2px solid #000000`
- Box shadow: `none`
- Hover: no transform, no box-shadow change — flat buttons do not lift in Win 3.1
- Hover background: `#D4D0C8` (very slight lightening — acceptable but optional)
- Active/pressed: invert border colors to simulate flat depression (optional Win 3.1 fidelity)
- Disabled: `pointer-events: none`, `opacity: 0.5`
- Size: inherit from existing `.button` (96x96px desktop, responsive per existing media queries)

Source: CONTEXT.md D-10, D-11; RESEARCH.md Pattern 1

### Program Manager (WIN31-04)

- Single window with Win 3.1 chrome (flat gray, navy title bar, 2px black border)
- Title bar text: "Program Manager"
- Menu bar: horizontal row below title bar — items: File, Options, Window, Help
  - File: `<span aria-disabled="true">File</span>` — visual only, not clickable
  - Options: `<div tabindex="0">` with `:focus-within` dropdown containing "Change Theme..." button
  - Window: `<span aria-disabled="true">Window</span>` — visual only
  - Help: `<span aria-disabled="true">Help</span>` — visual only
  - Menu bar `aria-label="Program Manager menu"`
  - "Change Theme..." calls `window.openThemeDialog()` on click
- Icon grid: 3 icons (About, Work, Contact), centered, horizontal layout with 24px gaps
- Each icon: 32x32 inline SVG + 11px label below
- Icon focus: dotted outline `1px dotted #000080`, navy highlight background `rgba(0,0,128,0.3)`
- Icon click: navigate to `/about`, `/work`, `/contact` (standard routing per D-07)
- CSS toggle: `.program-manager { display: none }` default; `[data-theme="win31"] .program-manager { display: flex }` active; `[data-theme="win31"] .retro-layout { display: none }`

Source: CONTEXT.md D-01 through D-07; RESEARCH.md Patterns 3, 4, 5

### Subpage Windows (WIN31-01, WIN31-04)

- Position: centered on desktop, `position: fixed`, `top: 50%; left: 50%; transform: translate(-50%, -50%)`
- Size: `width: min(95vw, 800px); height: min(90vh, 600px)` (inherits from existing `.content-window`)
- Chrome: same as window chrome above — flat gray, 2px black border, navy title bar
- Title bar shows page name ("About", "Work", "Contact")
- No menu bar on subpage windows (D-06)
- Home navigation: `[-]` control box in title bar links to `/` — decorated as anchor, not just span
- Desktop home icon also present (D-14), using existing `.desktop-icon` pattern styled for win31

Source: CONTEXT.md D-12, D-13, D-14; RESEARCH.md Pattern 6

## CSS Architecture Contract

### Token Override Structure

All win31 tokens defined as a single `[data-theme="win31"]` block in `src/styles/global.css`, immediately after the `[data-theme="retro"]` block. All component overrides scoped under `[data-theme="win31"] .component-class`.

### Required Token Overrides

```css
[data-theme="win31"] {
  --color-bg:                    #008080;   /* same teal desktop (D-19) */
  --color-window:                #C0C0C0;
  --color-button-bg:             #C0C0C0;
  --color-button-text:           #000000;
  --color-button-border:         #000000;
  --color-link:                  #008000;
  --color-link-hover:            #006600;
  --color-title:                 #FFFFFF;
  --color-title-gradient-start:  #FFFFFF;   /* disables retro gradient */
  --color-title-gradient-end:    #FFFFFF;
  --color-title-shadow:          transparent;
  --color-title-glow-1:          transparent;
  --color-title-glow-2:          transparent;
  --color-scrollbar-track:       #C0C0C0;
  --border-scrollbar:            1px solid #000;
  --color-scrollbar-thumb:       #808080;
  --color-scrollbar-thumb-hover: #606060;
  --color-desktop-icon:          #FFFFFF;
  --color-desktop-icon-shadow:   rgba(0,0,0,0.7);
  --color-desktop-icon-highlight: rgba(0,0,128,0.3);
  --outline-desktop-icon:        1px dotted #000080;
  --color-desktop-label-stroke:  #000;
  --filter-desktop-icon:         none;
  --font-body:                   "W95FA", Arial, sans-serif;
  --font-title:                  "W95FA", Arial, sans-serif;
  --border-button:               2px solid #000000;
  --border-headshot:             2px solid #000000;
  --shadow-button:               none;
  --shadow-button-hover:         none;
  --shadow-headshot:             none;
  --shadow-window:               none;
  --bg-gradient-fade:            transparent;
  --bg-pixel-light:              transparent;
  --bg-pixel-dark:               transparent;
  --bg-splash:                   #C0C0C0;
  --bg-content-window:           #C0C0C0;
}
```

### Required Body Override

The `body` element uses hardcoded `background-image` (not a token) — must override directly:

```css
[data-theme="win31"] body {
  background-image: none;  /* removes retro pixel-grid pattern */
  background-color: #008080;  /* solid teal desktop */
}
```

### Anti-Patterns

- Never use `border-style: outset` or `inset` in win31 rules — that is Win 95 (Phase 4)
- Never apply gradients — override all retro gradient references with solid colors
- Never scope rules as `:root[data-theme="win31"]` — project uses `[data-theme="win31"]` (no `:root`)
- Never leave `.window-title` transform in place — must be `transform: none` under win31
- Never render Program Manager or retro layout only — both must exist in DOM; CSS controls visibility

Source: RESEARCH.md Anti-Patterns, Pitfall 1, Pitfall 3

## Interaction Contract

### Menu Bar (Options Dropdown)

- Trigger: `:focus-within` on `.menubar-item[data-item="options"]` — CSS-only reveal
- Dismiss: clicking outside (native focus loss) or Escape (JS listener on keydown)
- "Change Theme..." click: calls `window.openThemeDialog()` (existing global)
- File, Window, Help: `aria-disabled="true"`, `tabindex="-1"` — keyboard skipped

### Icon Navigation

- Click: navigate to `/about`, `/work`, `/contact`
- Enter/Space: same as click (standard anchor behavior)
- Tab order: File (skipped) → Options → Window (skipped) → Help (skipped) → About icon → Work icon → Contact icon
- Focus visible: `outline: 1px dotted #000080` on icon container

### Title Bar Home Navigation

- `[-]` on subpage windows: anchor `href="/"` — navigates home on click
- Screen reader: `aria-label="Close - return to home"` on the anchor
- Desktop icon: `href="/"` anchor with home icon SVG — same behavior as existing HomeIcon.astro

### Theme Switching

- When user switches TO win31: CSS instantly applies `[data-theme="win31"]` tokens + `.program-manager { display: flex }` + `.retro-layout { display: none }`
- When user switches AWAY: retro tokens restore, program manager hides, retro layout shows
- No JS needed for layout toggling — pure CSS data-theme-driven

### View Transitions

- `.splash-screen` has existing `view-transition-name: window` — accept that cross-theme transitions may differ visually
- Win31 subpage windows: use same `view-transition-name: window` for within-theme navigation consistency
- Cross-theme navigation (retro → win31 or vice versa during page nav): visual difference is acceptable per RESEARCH.md Pitfall 4
